Build an Image Search Abstraction Layer

Objective: Build a full stack JavaScript app that allows you to search for images.

User Story:

  1. I can get the image URLs, alt text and page urls for a set of images relating to a given search string.
  2. I can paginate through the responses by adding a ?offset=2 parameter to the URL.
  3. I can get a list of the most recently submitted search strings.

My implementation

API endpoint:

  • https://aromatic-iridium.glitch.me/api/imagesearch/

    Required parameter: q

    If omited error occured and the pixabay search engine send response (see below).

    Example usage: https://aromatic-iridium.glitch.me/api/imagesearch/?q=<your-search-string>

    Optional parameter: offset (minimum: 0, maximum: 90); Recommended step: 10

    Example usage: https://aromatic-iridium.glitch.me/api/imagesearch/?q=<your-search-string>&offset=<integer>

  • https://aromatic-iridium.glitch.me//api/latest/imagesearch

    No parameter required

    Example usage: https://aromatic-iridium.glitch.me//api/latest/imagesearch

Response:

JSON array with 10 objects:

Response key Description
context Source page on WEB, which provides the original image.
snippet A little informations about the image.
thumbnail Low resolution image.
url The URL of image with a maximum resolution.

JSON array with 11 objects:

I mainly use Custom Search Engine from Google but unfortunately the unlimited version is not free. If my API hit the daily usage limit the Search Engine send an error response. To bypass this issue I built another image search engine used the pixabay API. It is free for use but linking back is greatly appreciated and allows image authors to gain exposure.

If the response arrive with 11 object then the first object consist of an error key.

error Informations about the error. Mainly the daily limit exceeded
snippet A link to Pixabay is required and you may use any logo for this purpose
thumbnail A logo for pixabay. (see more thumbnail2-5)
url Another logo for pixabay. Choose whatever you want or creat your own...

The following response objects are same with the above but there are more one key:

author Name and link to the page of author on pixabay. Please place this link anywhere on your page if use them photo.
